# Hugo Paper site for profound.fyi — Design

**Date:** 2026-04-25
**Status:** Approved (pending spec review)

## Purpose

Stand up a professional landing page at `profound.fyi` using Hugo and the [hugo-paper](https://themes.gohugo.io/themes/hugo-paper/) theme. The site has a home page and a posts section for occasional writing. Hosted on Netlify, built on every push.

This spec covers the structural scaffold only. Content, copy, custom styling, and analytics are out of scope.

## Constraints

- Hugo v0.154.5+extended, already installed locally
- Go 1.25.6, already installed locally
- British English (`en-gb`)
- GitHub org: `profoundhq`
- Domain: `profound.fyi`

## Architecture

Hugo extended serves a static site. The `hugo-paper` theme is consumed as a Hugo Module, which delegates dependency tracking to Go modules and makes upgrades a single `hugo mod get -u`. Netlify builds the site on every push to the default branch and serves the contents of `public/`.

## Project layout

```
profound.fyi/
├── .gitignore               # /public, /resources, .hugo_build.lock
├── hugo.toml                # site config + theme module import
├── go.mod                   # Hugo module manifest
├── go.sum                   # generated by hugo mod get
├── netlify.toml             # build pinning
├── archetypes/
│   └── default.md
├── content/
│   ├── _index.md            # home page (empty)
│   └── posts/
│       └── _index.md        # posts list page
├── static/                  # robots.txt, images, favicons
├── docs/
│   └── superpowers/specs/   # this file lives here
└── README.md                # local dev + deploy notes
```

No `themes/` directory — the theme is pulled in as a module under `_vendor/` only when `hugo mod vendor` runs (we won't run it).

## Configuration

### `hugo.toml`

```toml
baseURL = 'https://profound.fyi/'
languageCode = 'en-gb'
title = 'profound.fyi'
pagination.pagerSize = 10

[module]
  [[module.imports]]
    path = 'github.com/nanxiaobei/hugo-paper'

[menu]
  [[menu.main]]
    name = 'Posts'
    pageRef = '/posts'
    weight = 10

[params]
  mode = 'auto'        # respects OS dark/light preference
  author = 'Benedict Steele'
  # social = [...]     # filled in later
```

### `netlify.toml`

```toml
[build]
  publish = "public"
  command = "hugo --gc --minify"

[build.environment]
  HUGO_VERSION = "0.154.5"
  HUGO_VERSION_EXTENDED = "true"
  GO_VERSION = "1.25.6"
```

Pinning matters: Hugo modules need Go on the build server, and a Hugo version mismatch can break builds silently.

### `.gitignore`

```

/resources/
.hugo_build.lock

```

## Build workflow

The scaffolding sequence run locally:

1. `git init` in `/Users/benedictsteele/Development/profound.fyi`
2. Write `.gitignore` first so generated artefacts never get tracked
3. `hugo new site . --force` to scaffold into the empty directory
4. `hugo mod init github.com/profoundhq/profound.fyi`
5. Write `hugo.toml` with the module import
6. `hugo mod get github.com/nanxiaobei/hugo-paper`
7. `hugo mod tidy`
8. Write `netlify.toml`, `README.md`, `content/_index.md`, `content/posts/_index.md`
9. Run `hugo server` and verify the home page renders with the Paper theme
10. Stop the server, commit the baseline

## Deployment (manual, user-driven)

These steps are not automated by this spec — Benedict performs them once the local scaffold is committed:

- Create `profound.fyi` repo under the `profoundhq` GitHub org
- `git remote add origin` and push `main`
- In Netlify: New site → import from GitHub → select `profoundhq/profound.fyi`. `netlify.toml` is auto-detected
- Add custom domain `profound.fyi` in Netlify and update DNS

## Verification

The scaffold is complete when:

- `hugo server` starts cleanly with no module errors
- The home page at `http://localhost:1313/` renders using the Paper theme (visible header, theme switcher, no "no theme" warning)
- The `/posts/` page renders an empty post list without errors
- A clean `hugo --gc --minify` produces a `public/` directory containing `index.html` referencing theme assets

## Out of scope

- Content and copy
- Custom CSS, partials, or layout overrides
- Social links and avatar
- Analytics, comments, search
- CI beyond Netlify's own build pipeline
- `hugo mod vendor` (not needed for Netlify builds)

## Risks and notes

- **Theme upstream changes.** `hugo-paper` is a third-party repo. Pinning the module version on first `hugo mod get` is automatic; upgrades are explicit.
- **Netlify Go version drift.** If Netlify changes its default Go, our pin protects us. The pinned versions need bumping when local toolchain moves on.
- **Module path is not the repo URL yet.** `github.com/profoundhq/profound.fyi` is declared before the repo exists on GitHub. This is fine — Hugo only resolves the path when fetching a module *from* it, which we never do for the site itself.
